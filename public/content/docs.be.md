# Змест

# Памылкі

## Паведамленне ніхто не атрымлівае

Памылка ўзнікае, калі скрыпт адпраўляе паведамленне, якое не атрымлівае ні адзін са скрыптаў. Магчыма, вы забыліся дадаць атрыманне гэтага паведамлення або выбралі атрыманне іншага паведамлення.

**Прыклад памылкі:**

```scratch
when green flag clicked
broadcast [level_start v]
```

*(У праекце няма ніводнага блока `scratch:when I receive [level_start v]`, таму гэтае паведамленне «ідзе ў пустату»).*

**Як выправіць?**
Трэба пераканацца, што ў блоках-трыгерах `scratch:when I receive [... v]` выбрана правільнае імя паведамлення, якое адпавядае блоку `scratch:broadcast [... v]`.

## Неадпраўленае паведамленне

Гэтая памылка з'яўляецца зваротнай да папярэдняй. Скрыпт настроены на чаканне паведамлення, але ні адзін скрыпт у праекце не адпраўляе яго. З-за гэтага код унутры гэтай падзеі ніколі не выканаецца.

**Прыклад памылкі:**

```scratch
when I receive [game_end v]
stop [all]
```

*(Калі ў праекце нідзе не выкарыстоўваецца блок `scratch:broadcast [game_end v]`, гульня ніколі не спыніцца).*

**Як выправіць?**
Праверце ўсе блокі `scratch:broadcast [... v]` у праекце. Пераканайцеся, што імя паведамлення ў іх супадае з імем у блоку чакання. Калі паведамленне больш не патрэбнае, выдаліце блок `scratch:when I receive [... v]`.

## Пераменная без пачатковага значэння

Пераменная выкарыстоўваецца ў скрыпце, але ёй не прысвойваецца пачатковае значэнне пры запуску праекта. Гэта можа прывесці да таго, што ў новай гульні захаваюцца ачкі або становішча гульца з мінулай сесіі.

**Прыклад памылкі:**

```scratch
when green flag clicked
forever
  if <touching [edge v]?> then
    change (score::variables) by (1)
  end
end
```

**Як выправіць?**
Дадайце блок `scratch:set (score::variables) to [0]` адразу пасля блока `scratch:when green flag clicked`.

```scratch
when green flag clicked
set (score::variables) to (0)
forever
  if <touching [edge v]?> then
    change (score::variables) by (1)
  end
end
```

## Сравненне літаральных значэнняў

Ва ўмове аператара `if` або `repeat until` параўноўваюцца два пастаянныя значэнні (лікі або тэкст), а не пераменныя. Вынік такога параўнання заўсёды аднолькавы.

**Прыклад памылкі:**

```scratch
if <(5) > (10)> then
  say [This will never happen]
end
```

**Як выправіць?**
Замяніце адно са значэнняў на пераменную або сэнсар. Напрыклад: `scratch:if <(score) > (10)> then`.

# Папярэджанні

## Пусты спрайт

У праекце ёсць спрайт, у якога няма ніводнага блока кода. Гэта лішні аб'ект, які займае памяць.

**Як выправіць?**
Калі спрайт павінен быць актыўным, дадайце яму скрыпты (напрыклад, `scratch:when green flag clicked`). Калі ён лішні — выдаліце яго.

## Неўжываная пераменная

У праекце створана пераменная, але яна не выкарыстоўваецца ні ў адным блоку.

**Як выправіць?**
Выдаліце пераменную `scratch:(my_variable)` у панэлі «Пераменныя», калі яна не патрэбная для адлюстравання на экране або логікі.

## Загублены код

Блокі кода знаходзяцца ў рабочай вобласці, але не прымацаваныя да «шапкі» (падзеі). Такі код «шэры» і не працуе.

**Прыклад заўвагі:**

```scratch
move (10) steps
next costume
```

*(Гэтыя блокі проста ляжаць у полі; над імі няма блока `scratch:when green flag clicked`).*

**Як выправіць?**
Прыцупуйце блокі да адпаведнай падзеі або выдаліце іх.

## Няма каментарыяў

У праекце адсутнічаюць тлумачальныя каментарыі. Гэта ўскладняе разуменне кода.

**Як выправіць?**
Націсніце правай кнопкай мышы на блок і выберыце «Дадаць каментарый».

```scratch
when green flag clicked // Initializing the game
set [lives] to (3)
```

## Накладанне скрыптаў

Блокі кода візуальна накладваюцца адзін на аднаго. Гэта не ўплывае на працу праграмы, але замінае вам яе чытаць.

**Як выправіць?**
Акуратна разнясіце скрыпты ў бакі або націсніце правай кнопкай мышы на пустым месцы ў вобласці скрыптоў і выберыце **«Ачысціць» (Clean up Blocks)**.

## Занадта доўгі скрыпт

Адзін скрыпт змяшчае дзясяткі блокаў. Такія «макароны» з кода складана выпраўляць.

**Як выправіць?**
Выкарыстоўвайце «Мае блокі», каб вынесці частку логікі ў асобную функцыю.

```scratch
define reset_player
go to x: (0) y: (0)
set [speed] to (0)
switch costume to [idle]

when green flag clicked
reset_player
```

## Стандартнае імя спрайта

Імёны накшталт `Sprite1` або `Спрайт1` не кажуць пра тое, што робіць аб'ект.

**Як выправіць?**
Перайменуйце спрайт у полі пад сцэнай. Напрыклад, замест `Sprite1` напішыце `Player` або `Ball`.
