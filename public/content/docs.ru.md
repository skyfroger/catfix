# Ошибки

## Сообщение никто не получает

Ошибка возникает, когда скрипт отправляет сообщение, которое не получает ни один из скриптов. Возможно, вы забыли добавить получение этого сообщения или выбрали получение другого сообщения.

**Пример ошибки:**

```scratch
when green flag clicked
broadcast [level_start v]
```

*(В проекте нет ни одного блока `scratch:when I receive [level_start v]`, поэтому это сообщение «уходит в пустоту»).*

**Как исправить?**
Нужно убедиться, что в блоках-триггерах `scratch:when I receive [... v]` выбрано правильное имя сообщения, соответствующее блоку `scratch:broadcast [... v]`.

## Неотправленное сообщение

Эта ошибка является обратной к предыдущей. Скрипт настроен на ожидание сообщения, но ни один скрипт в проекте не отправляет его. Из-за этого код внутри этого события никогда не выполнится.

**Пример ошибки:**

```scratch
when I receive [game_end v]
stop [all]
```

*(Если в проекте нигде не используется блок `scratch:broadcast [game_end v]`, игра никогда не остановится).*

**Как исправить?**
Проверьте все блоки `scratch:broadcast [... v]` в проекте. Убедитесь, что имя сообщения в них совпадает с именем в блоке ожидания. Если сообщение больше не нужно, удалите блок `scratch:when I receive [... v]`.

## Переменная без начального значения

Переменная используется в скрипте, но ей не присваивается начальное значение при запуске проекта. Это может привести к тому, что в новой игре сохранятся очки или положение игрока из старой сессии.

**Пример ошибки:**

```scratch
when green flag clicked
forever
  if <touching [edge v]?> then
    change (score::variables) by (1)
  end
end
```

**Как исправить?**
Добавьте блок `scratch:set (score::variables) to [0]` сразу после блока `scratch:when green flag clicked`.

```scratch
when green flag clicked
set (score::variables) to (0)
forever
  if <touching [edge v]?> then
    change (score::variables) by (1)
  end
end
```

## Сравнение буквальных значений

В условии оператора `if` или `repeat until` сравниваются два постоянных значения (числа или текст), а не переменные. Результат такого сравнения всегда одинаков.

**Пример ошибки:**

```scratch
if <(5) > (10)> then
  say [This will never happen]
end
```

**Как исправить?**
Замените одно из значений на переменную или сенсор. Например: `scratch:if <(score) > (10)> then`.

# Замечания

## Пустой спрайт

В проекте есть спрайт, у которого нет ни одного блока кода. Это лишний объект, который занимает память.

**Как исправить?**
Если спрайт должен быть активным, добавьте ему скрипты (например, `scratch:when green flag clicked`). Если он лишний — удалите его.

## Неиспользуемая переменная

В проекте создана переменная, но она не используется ни в одном блоке.

**Как исправить?**
Удалите переменную `scratch:(my_variable)` в панели «Переменные», если она не нужна для отображения на экране или логики.

## Потерянный код

Блоки кода находятся в рабочей области, но не прикреплены к «шапке» (событию). Такой код «серый» и не работает.

**Пример замечания:**

```scratch
move (10) steps
next costume
```

*(Эти блоки просто лежат в поле, над ними нет блока `scratch:when green flag clicked`).*

**Как исправить?**
Прикрепите блоки к соответствующему событию или удалите их.

## Нет комментариев

В проекте отсутствуют поясняющие комментарии. Это затрудняет понимание кода.

**Как исправить?**
Нажмите правой кнопкой мыши на блок и выберите «Добавить комментарий».

```scratch
when green flag clicked // Initializing the game
set [lives] to (3)
```

## Наложение скриптов

Блоки кода визуально накладываются друг на друга. Это не влияет на работу программы, но мешает вам её читать.

**Как исправить?**
Аккуратно разнесите скрипты в стороны или нажмите правой кнопкой мыши на пустом месте в области скриптов и выберите **«Очистить» (Clean up Blocks)**.

## Слишком длинный скрипт

Один скрипт содержит десятки блоков. Такие «макароны» из кода сложно исправлять.

**Как исправить?**
Используйте «Мои блоки», чтобы вынести часть логики в отдельную функцию.

```scratch
define reset_player
go to x: (0) y: (0)
set [speed] to (0)
switch costume to [idle]

when green flag clicked
reset_player
```

## Стандартное имя спрайта

Имена вроде `Sprite1` или `Спрайт1` не говорят о том, что делает объект.

**Как исправить?**
Переименуйте спрайт в поле под сценой. Например, вместо `Sprite1` напишите `Player` или `Ball`.
